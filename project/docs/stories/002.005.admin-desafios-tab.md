## Dev Notes

**Implementation Order:**
1. Check if challenges data exists or needs backend functions
2. Create DesafiosSection component with challenges list
3. Create edit/delete modals
4. Create category management UI
5. Set up real-time listener for updates
6. Add form validation

**Type Definitions:**
```typescript
interface Challenge {
  id: string
  name: string
  description: string
  category: string
  points: number
  difficulty: 'fácil' | 'médio' | 'difícil'
  is_active: boolean
  created_at: string
}

interface ChallengeCategory {
  id: string
  name: string
  color: string
  challenge_count: number
  total_points: number
}

interface ChallengeInput {
  name: string
  description: string
  category: string
  points: number
  difficulty: 'fácil' | 'médio' | 'difícil'
}
```

**Real-Time Listeners:**
- Listen to challenges INSERT for new challenges
- Listen to challenges UPDATE for edits
- Listen to challenge_categories for changes

**Error Handling:**
- Failed delete → show error toast, don't update UI
- Failed create → show error toast, preserve form data
- Failed category delete with active challenges → show validation error
- Listener errors → log and retry
