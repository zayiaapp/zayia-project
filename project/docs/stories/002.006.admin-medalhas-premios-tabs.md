## Problem Statement

EPIC-001 created the medals and prizes database. Now we need:

- ❌ No admin UI to view all medals/badges
- ❌ No way to create new medals
- ❌ No way to edit existing medals
- ❌ No way to delete medals
- ❌ No way to manage prizes (rewards)
- ❌ No way to configure when medals unlock (thresholds)
- ❌ No visibility into which users have earned medals/prizes

**This story creates admin Medalhas & Prêmios tabs with full CRUD for medals and prizes.**

**Type Definitions:**
```typescript
interface Medal {
  id: string
  name: string
  description: string
  icon: string
  rarity: 'comum' | 'raro' | 'épico' | 'lendário'
  unlock_condition?: string
  unlock_threshold?: number
  is_active: boolean
  created_at: string
}

interface Prize {
  id: string
  name: string
  description: string
  type: 'badge' | 'points' | 'discount'
  value: number
  medal_id?: string
  is_active: boolean
  created_at: string
}

interface UnlockRule {
  id: string
  medal_id: string
  condition: 'challenges_completed' | 'points_earned' | 'level_reached'
  threshold: number
  scope: 'global' | 'category'
}
```

**Rarity Colors:**
- Comum: Gray/Blue
- Raro: Purple
- Épico: Gold/Orange
- Lendário: Rainbow/Gradient

**Real-Time Listeners:**
- Listen to medals INSERT for new medals
- Listen to medals UPDATE for changes
- Listen to prizes INSERT/UPDATE
- Listen to user_medals for earned medals

**Error Handling:**
- Failed delete → show error toast
- Failed create → show error toast, preserve form data
- Failed unlock rule update → show validation error
- Listener errors → log and retry
