## Problem Statement

EPIC-001 created the community API. Now we need:

- ❌ No admin UI to view all community messages in real-time
- ❌ No way to view and edit community rules
- ❌ No way to delete inappropriate messages
- ❌ No way to ban users from the community
- ❌ No ban auto-expiry logic (1d/7d/permanent)
- ❌ No real-time sync of moderation actions

**This story creates the admin Community tab with full moderation tools.**

## Dev Notes

**Implementation Order:**
1. Check if admin community section exists or needs creation
2. Implement real-time message display with pagination
3. Create edit rules modal
4. Create delete message modal with reasons
5. Create ban user modal with duration
6. Set up real-time listeners for all updates

**Type Definitions:**
```typescript
interface DeleteReason {
  value: 'disrespectful' | 'inappropriate' | 'spam' | 'discrimination' | 'privacy' | 'other'
  label: string
}

interface BanDuration {
  value: '1_day' | '7_days' | 'permanent'
  label: string
}
```

**Real-Time Listeners:**
- Listen to community_messages INSERT for new messages
- Listen to community_messages UPDATE for edits/deletions
- Listen to community_bans INSERT for bans
- Listen to community_rules UPDATE for rule changes

**Error Handling:**
- Failed delete → show error toast, don't update UI
- Failed ban → show error toast, check if user already banned
- Failed rules update → show error toast, preserve old rules
- Listener errors → log and retry
