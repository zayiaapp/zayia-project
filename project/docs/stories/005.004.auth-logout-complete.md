## Problem Statement

O logout atual em `AuthContext.tsx` chama `supabase.auth.signOut()`, mas **não limpa todo o estado local**:

1. `localStorage` ainda contém:
   - `user_points` (pontos do usuário)
   - `user_points_history` (histórico de pontos)
   - `zayia_user` (dados do usuário)
   - `zayia_profile` (perfil)
   - `zayia_user_challenges_*` (desafios completados por userId)
   - `zayia_integrations` (configurações de integrações)
   - `earned_badges_*` (badges ganhos — localStorage de medals)
   - `user_streak` (streak atual)

2. Após logout + login com **outra conta**, o estado antigo pode vazar para o novo usuário
3. Se o browser recarregar antes do logout completar, estado inconsistente fica no localStorage

## Acceptance Criteria

### AC1: Logout limpa todo o localStorage
- [ ] Encontrar o método `signOut()` em `src/contexts/AuthContext.tsx`
- [ ] Substituir/expandir para:

```typescript
const signOut = async () => {
  try {
    // 1. Encerra sessão no Supabase Auth
    await supabase.auth.signOut()
  } catch (err) {
    console.error('❌ Erro no signOut Supabase:', err)
    // Continua mesmo com erro — limpeza local é crítica
  } finally {
    // 2. Limpa TODAS as keys do localStorage relacionadas ao usuário
    const keysToRemove = [
      'user_points',
      'user_points_history',
      'zayia_user',
      'zayia_profile',
      'zayia_integrations',
    ]
    keysToRemove.forEach(key => localStorage.removeItem(key))

    // 3. Limpa keys dinâmicas (padrão: zayia_user_challenges_*)
    const allKeys = Object.keys(localStorage)
    allKeys.forEach(key => {
      if (
        key.startsWith('zayia_user_challenges_') ||
        key.startsWith('earned_badges_') ||
        key.startsWith('user_streak')
      ) {
        localStorage.removeItem(key)
      }
    })

    // 4. Limpa estado React
    setUser(null)
    setProfile(null)

    console.log('✅ Logout completo — localStorage limpo')
  }
}
```

## Risks & Blockers

### Risk 1: `signOut()` falha silenciosamente (Baixo)
- **Problema:** Se a chamada `supabase.auth.signOut()` falhar (sem internet), o usuário fica logado no Supabase Auth mas o estado local foi limpo.
- **Mitigação:** Use `try/catch/finally` — o bloco `finally` limpa o estado mesmo se a chamada falhar. Na próxima navegação, o `getSession()` vai retornar null.
- **Impacto:** UX aceitável — usuário parece deslogado mas a sessão Supabase expira naturalmente.

### Risk 2: Keys dinâmicas de localStorage não são todas conhecidas (Médio)
- **Problema:** O padrão `zayia_user_challenges_*` assume um userId sufixo. Se o padrão mudar, keys ficam no localStorage.
- **Mitigação:** Usar `Object.keys(localStorage).filter(k => k.startsWith(...))` como no código acima — captura qualquer variação do padrão.
- **Como verificar:** Abrir DevTools → Application → Local Storage → ver keys antes e depois do logout.
