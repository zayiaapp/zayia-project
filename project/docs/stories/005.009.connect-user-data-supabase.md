  const { data } = await supabase

### AC6: Histórico de pontos dos últimos 7 dias (melhoria)

- [ ] Implementar query para pontos por dia:
```typescript
async getPointsHistory(userId: string, days = 7): Promise<Array<{date: string; points: number}>> {
  const since = new Date(Date.now() - days * 24 * 60 * 60 * 1000).toISOString()
  const { data } = await supabase
    .from('challenge_completions')
    .select('points_earned, completed_at')
    .eq('user_id', userId)
    .gte('completed_at', since)
    .order('completed_at', { ascending: true })

  // Agrupa por dia
  const byDay: Record<string, number> = {}
  data?.forEach(c => {
    const day = c.completed_at.split('T')[0]
    byDay[day] = (byDay[day] || 0) + (c.points_earned || 0)
  })

  // Gera array com todos os dias (zeros onde não houve atividade)
  const result = []
  for (let i = days; i >= 0; i--) {
    const d = new Date(Date.now() - i * 24 * 60 * 60 * 1000)
    const dayStr = d.toISOString().split('T')[0]
    result.push({ date: dayStr, points: byDay[dayStr] || 0 })
  }
  return result
}
```

## Risks & Blockers

### Risk 1: Realtime de pontos pode ter latência (Baixo)
- **Problema:** Após completar desafio, pode haver 1-2s antes do header atualizar via realtime.
- **Mitigação:** Isso é aceitável. Se quiser mais rápido, pode atualizar o profile state local otimisticamente.
- **Impacto:** UX levemente menos instantânea, mas correta.

### Risk 2: `getUserRanking()` — N usuários com mesmo número de pontos (Baixo)
- **Problema:** Se 3 usuários têm 100 pontos, `gt('points', 100)` retorna 0 — todos são "1º lugar".
- **Mitigação:** Para a escala atual é OK. Futuramente pode usar `gte` + desempate por `created_at`.

### Risk 3: `challenge_completions` pode não ter `points_earned` preenchido (Médio)
- **Problema:** Se as completions antigas (8 existentes) não têm `points_earned`, `getTodayStats()` retorna 0.
- **Mitigação:** Verificar schema: `SELECT column_name FROM information_schema.columns WHERE table_name = 'challenge_completions'`
- **Impacto:** Histórico antigo zerado, mas novos registros vão ter pontos corretos.
