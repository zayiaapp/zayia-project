## Problem Statement

Esta é a story de validação final. Após todas as migrações, precisamos garantir que o fluxo completo de uma nova usuária funciona do zero ao fim sem nenhum erro.

Fluxo a validar:
1. Criar conta (signup)
2. Fazer login
3. Selecionar categoria de desafios
4. Completar desafio fácil
5. Completar desafio difícil
6. Ganhar medalha
7. Ver posição no ranking
8. Postar na comunidade
9. Editar/deletar post
10. Fazer logout
11. Fazer login novamente → dados persistem

## Acceptance Criteria

### AC1: Roteiro completo de teste — passo a passo

O dev deve executar CADA passo abaixo, anotar o resultado, e marcar ✅ ou ❌:

#### FASE 1: Conta Nova

```
Passo 1: Abrir http://localhost:5173 em browser ANÔNIMO (sem cookies/localStorage)
  Esperado: Tela de login aparece (AuthPage)
  Verificar: Console sem erros JavaScript

Passo 2: Clicar em "Criar conta" (ou navegar para SignUp)
  Esperado: Formulário de cadastro aparece

Passo 3: Preencher: Nome, Email novo (não cadastrado), Senha >= 6 chars
  Esperado: Formulário aceita os dados
  Verificar: Validações funcionam (email inválido → erro, senha curta → erro)

Passo 4: Clicar em "Criar conta"
  Esperado: Login automático OU mensagem de "verifique seu email"
  Verificar: profiles table no Supabase → nova row criada com role='user'
  Verificar: Console sem erros

Passo 5: Se exigir verificação de email, verificar inbox e clicar no link
  Esperado: Redireciona para o app logado
```

#### FASE 2: Login e Estado Inicial

```
Passo 6: Verificar Dashboard inicial
  Esperado:
    - Nome correto no header
    - Pontos: 0
    - Nível: 1
    - Streak: 0
    - Seção Desafios: Modal de seleção de categoria aparece (sem categoria ativa)

Passo 7: Verificar console
  Esperado: Nenhum erro. Avisos aceitáveis (ex: deprecated warnings)
  Não aceitável: "undefined is not a function", "Cannot read properties of null", etc.
```

#### FASE 3: Completar Desafios

```
Passo 8: Selecionar categoria no modal (ex: Autoestima)
  Esperado: Modal fecha, lista de desafios carrega
  Verificar: Desafios da categoria Autoestima aparecem (60 fáceis + 60 difíceis = 120)

Passo 9: Completar 1 desafio FÁCIL
  a. Clicar "Começar Desafio" em qualquer desafio fácil
  b. Timer de 60s inicia (modal fullscreen)
  c. Aguardar timer → selecionar arquivo de foto/vídeo → clicar "Enviar"
  Esperado:
    - Desafio marcado como ✅ Concluído
    - challenge_completions: nova row no Supabase
    - profiles.points: +10 pontos no Supabase
    - Header mostra novos pontos (via realtime, < 3s)
    - dailyProgressUpdated event dispara (barra de progresso atualiza)

Passo 10: Completar 1 desafio DIFÍCIL
  Esperado:
    - +25 pontos
    - profiles.points: 10 + 25 = 35 no Supabase

Passo 11: Verificar barra de progresso
  Esperado: 2/120 desafios completados (não 2/20 ou 2/999)
```

#### FASE 4: Medalha

```
Passo 12: Completar desafios suficientes para destravar medalha
  (Ex: primeira medalha pode ser após 5 desafios — verificar BADGES array)
  Esperado:
    - PopUpMedalUnlocked aparece com ícone e nome da medalha
    - user_earned_badges: nova row no Supabase
    - Botão "Ver Medalha" funciona → navega para aba de medalhas
    - Medalha aparece como desbloqueada em MedalsSection

Passo 13: Recarregar página (F5)
  Esperado: Medalha AINDA aparece como desbloqueada (prova de persistência no Supabase)
```

#### FASE 5: Ranking

```
Passo 14: Ir para seção Ranking
  Esperado:
    - Posição atual exibida (ex: "1º lugar" se for o único usuário)
    - Leaderboard com usuários reais (não mock)
    - Pontos exibidos batem com profiles.points no Supabase
```

#### FASE 6: Comunidade

```
Passo 15: Ir para seção Comunidade
  Esperado: Lista de mensagens carrega (pode ser vazia)
  Verificar: Console sem erros

Passo 16: Postar mensagem: "Primeira mensagem de teste!"
  Esperado:
    - Mensagem aparece imediatamente na lista (otimismo)
    - community_messages: nova row no Supabase
    - Se abrir em outro browser: mensagem aparece em < 2s (realtime)

Passo 17: Deletar a mensagem postada
  Esperado:
    - Mensagem desaparece da lista
    - No Supabase: row tem deleted_at preenchido (soft delete) OU row removida

Passo 18: Postar nova mensagem com emoji: "Olá! 🌟"
  Esperado: Mensagem aceita emoji, aparece corretamente
```

#### FASE 7: Logout e Re-login

```
Passo 19: Clicar em Logout
  Esperado:
    - Volta para AuthPage
    - Abrir DevTools → Application → Local Storage → keys zayia_* removidas

Passo 20: Login com as mesmas credenciais do Passo 3
  Esperado:
    - Dashboard carrega com os MESMOS dados de antes
    - Pontos: 35 (10 + 25 do Passo 10)
    - Medalha: ainda desbloqueada
    - Categoria: Autoestima (ativa)
    - Mensagem "Olá! 🌟" ainda visível na comunidade
    - NENHUM dado foi perdido
```

## Risks & Blockers

### Risk 1: Fluxo de signup pode exigir verificação de email (Médio)
- **Problema:** Se Supabase estiver configurado para confirmar email, o novo usuário precisa verificar o email antes de logar.
- **Verificação:** Supabase Dashboard → Authentication → Settings → "Enable Email Confirmations"
- **Mitigação para teste:** Desabilitar temporariamente a confirmação de email para facilitar o teste. OU usar o email de um usuário já verificado.

### Risk 2: Medalha pode não destravar com poucos challenges (Baixo)
- **Problema:** Não sabemos exatamente quantos challenges disparam a primeira medalha.
- **Mitigação:** Verificar `BADGES` array em `badges-data-mock.ts` para encontrar a medalha de menor requisito.
- **Alternativa para teste:** Inserir manualmente na tabela `user_earned_badges` via Supabase Dashboard para simular uma medalha desbloqueada.

### Risk 3: Arquivos de prova (upload) podem falhar em localhost (Baixo)
- **Problema:** O upload de foto usa Supabase Storage. A bucket pode não estar configurada para localhost.
- **Verificação:** Supabase Dashboard → Storage → verificar se a bucket `challenge-proofs` existe e se tem permissões para upload autenticado.
