## Problem Statement

ZAYIA currently operates with a **hybrid architecture**: some data in Supabase, most critical data in localStorage/JSON. This creates:

- ❌ Data loss on app uninstall
- ❌ No multi-device sync (user sees different data on phone vs web)
- ❌ Auth uses lookup table, not Supabase native auth
- ❌ Challenges, badges, rankings, community live in localStorage
- ❌ No real-time sync for live features
- ❌ No persistent state for subscription tracking
- ❌ Stripe webhooks have nowhere to persist data

**This story fixes the foundation: Supabase Auth + Core Tables (Challenges, Badges).**

### Task 1.7: RLS Policies + Security
**Owner:** @dev / @data-engineer
**Effort:** 2 points

**Migration includes:**
```sql
-- Public read for challenges
ALTER TABLE challenges ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Challenges are public"
  ON challenges FOR SELECT USING (true);

-- Badges public read
ALTER TABLE badges ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Badges are public"
  ON badges FOR SELECT USING (true);

-- user_earned_badges: users see own, can insert own
ALTER TABLE user_earned_badges ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users see own badges"
  ON user_earned_badges FOR SELECT
  USING (auth.uid() = user_id);
CREATE POLICY "Users can earn badges"
  ON user_earned_badges FOR INSERT
  WITH CHECK (auth.uid() = user_id);
```

**Acceptance:**
- [ ] Policies created for all new tables
- [ ] Tested: anon user can read challenges
- [ ] Tested: auth user can see only own badges

## Dev Notes

**Current Architecture:**
- Auth: lookup table, not native Supabase auth
- Challenges: JSON files + localStorage
- Badges: localStorage only
- No real-time sync

**After Phase 1:**
- Auth: Supabase native (secure, scalable)

- Badges: Supabase table + real-time listeners (Phase 2)
- Foundation for real-time sync, subscriptions, rankings
