```typescript
function formatRelativeTime(isoString: string): string {
  const seconds = Math.floor((Date.now() - new Date(isoString).getTime()) / 1000)
  if (seconds < 60) return 'agora'
  if (seconds < 3600) return `há ${Math.floor(seconds / 60)} min`
  if (seconds < 86400) return `há ${Math.floor(seconds / 3600)}h`
  return `há ${Math.floor(seconds / 86400)}d`
}
```

## Risks & Blockers

### Risk 1: `activity_log` vazio inicialmente (Baixo)
- **Problema:** Se STORY-005.001 foi feita mas ainda nenhum challenge foi completado, `activity_log` está vazio.
- **Mitigação:** UI mostra "Nenhuma atividade recente" — isso é o comportamento correto.
- **Impacto:** Zero — é o estado inicial esperado.

### Risk 2: `monthly_rankings` pode estar vazio (Baixo)
- **Problema:** Se a tabela `monthly_rankings` não foi populada ainda, `allWinners` fica vazio.
- **Mitigação:** AnalyticsSection mostra mensagem "Nenhum vencedor registrado ainda."
- **Verificar:** `SELECT COUNT(*) FROM monthly_rankings` antes de implementar.

### Risk 3: TypeScript quebra quando Profile muda (Médio)
- **Problema:** `rankingUsers` era `any[]` (de generateMockRankingUsers) — agora é `Profile[]`. Se o JSX que renderiza `rankingUsers` acessar propriedades que não existem em Profile, TypeScript vai reclamar.
- **Mitigação:** O dev deve ver os erros de build e corrigir campo a campo. Não é bloqueante, só trabalhoso.
